Кадровые уведомления 1С:ЗУП
===========================

Учебная конфигурация на платформе **1С:Предприятие 8.3** для автоматизации
процесса регистрации кадровых изменений и контроля email-уведомлений
ответственных лиц.

Проект выполнен в рамках преддипломной практики по направлению подготовки
**09.03.03 «Прикладная информатика»**, профиль **«Корпоративные информационные
системы»**.

Назначение проекта
------------------

Конфигурация предназначена для демонстрации автоматизации процесса кадрового
уведомления.

Рассматриваемый процесс:

**Регистрация кадрового изменения и уведомление ответственных лиц.**

В исходном варианте процесс выполняется вручную: кадровик получает основание
кадрового изменения, подготавливает данные сотрудника, вручную определяет
получателей, отправляет сообщение через отдельный канал связи и разрозненно
фиксирует результат. Это затрудняет контроль статусов, ошибок отправки и сводной
аналитики.

Разработанная конфигурация связывает кадровое изменение, получателей
уведомлений, статусы обработки, регистры накопления и отчеты в едином контуре
1С.

Основной сценарий работы
------------------------

1.  Кадровик создает документ **«Кадровое изменение»**.

2.  В документе указывается сотрудник, вид изменения, подразделение, должность,
    оклад и основание.

3.  При необходимости заполняются получатели уведомления.

4.  После проведения документа формируются движения по регистрам накопления.

5.  Система фиксирует кадровое событие и данные по уведомлениям.

6.  Пользователь может сформировать отчеты по кадровым изменениям, уведомлениям
    и ошибкам отправки.

Порядок проверки проекта
------------------------

### 1. Проверка конфигурации

1.  Открыть платформу **1С:Предприятие 8.3**.

2.  Создать новую информационную базу.

3.  Загрузить файл `Lubomirov_Kirill_HR_Email_Notifications.dt`.

4.  Открыть базу в режиме **Конфигуратор**.

5.  Выполнить проверку конфигурации.

6.  Убедиться, что основные объекты метаданных присутствуют.

### 2. Проверка пользовательского сценария

1.  Открыть базу в режиме **1С:Предприятие**.

2.  Перейти в подсистему **Кадровые изменения**.

3.  Создать документ **«Кадровое изменение»**.

4.  Заполнить сотрудника, вид изменения, дату, подразделение, должность и
    основание.

5.  Указать необходимость уведомления.

6.  Заполнить получателей уведомления.

7.  Провести документ.

8.  Проверить движения по регистрам накопления.

9.  Сформировать отчеты по кадровым изменениям и уведомлениям.

### 3. Проверка отчетов

Проверяются следующие отчеты:

-   кадровые изменения за период;

-   отправленные уведомления за период;

-   ошибки отправки уведомлений;

-   статистика по видам кадровых изменений;

-   уведомления по подразделениям.

Отчеты должны формироваться на тестовых данных и отображать результаты
проведения документов.
