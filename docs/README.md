# Документация проекта

Проект: **Кадровые уведомления 1С:ЗУП**  
Файл конфигурации: `Lubomirov_Kirill_HR_Email_Notifications.cf`  
Файл информационной базы: `Lubomirov_Kirill_HR_Email_Notifications.dt`

## Краткое описание

Конфигурация предназначена для учебного моделирования процесса регистрации кадровых изменений и связанных email-уведомлений. В системе реализованы справочники кадрового контура, документы, регистры накопления, отчёты, роли, пользователи, задача и бизнес-процесс.

## Функциональные блоки

### Кадровые изменения

Блок содержит данные и документы, связанные с изменением состояния сотрудника: переводом, изменением должности, изменением оклада, изменением подразделения, приёмом и увольнением.

Основной объект блока — документ «Кадровое изменение».

### Email-уведомления

Блок содержит получателей, шаблоны писем, документы уведомлений и регистр отправленных уведомлений. Реальная SMTP-отправка не выполняется. Система работает в учебном тестовом режиме и фиксирует результат отправки в регистре.

### Правила рассылки

Блок связывает вид кадрового изменения, группу получателей, шаблон письма и приоритет уведомления.

### Зарплата и персонал

Блок содержит справочные кадровые данные: сотрудников, должности и подразделения.

### Отчёты и контроль

Блок содержит отчёты:

1. Кадровые изменения за период.
2. Отправленные уведомления за период.
3. Ошибки отправки уведомлений.
4. Статистика по видам кадровых изменений.
5. Уведомления по подразделениям.

### Администрирование

Блок содержит пользователей, роли и технические настройки, включая SMTP-серверы.

## Основной сценарий

1. Кадровик создаёт документ «Кадровое изменение».
2. Заполняет данные сотрудника и параметры изменения.
3. Добавляет получателей уведомления.
4. Проводит документ.
5. Система записывает движения по регистрам.
6. Пользователь формирует отчёты.

## Результат проведения документа

При корректном проведении документа система выводит сообщение:

```text
Проведение выполнено. Кадровые события: 1, уведомления: 2
```

Это означает, что создана одна запись кадрового события и две записи уведомлений по получателям.

## Бизнес-процесс

Бизнес-процесс «Обработка кадрового изменения» описывает маршрут:

```text
Старт → Проверка кадровиком → Формирование уведомления → Отправка уведомления → Контроль результата → Повторная отправка → Завершение
```

Он используется как логическая модель обработки кадрового изменения и подтверждается картой маршрута в конфигураторе.
